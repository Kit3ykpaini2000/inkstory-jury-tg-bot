# inkstory-bot

Telegram-бот для организации литературного конкурса на [inkstory.net](https://inkstory.net).  
Парсит посты участников, распределяет их по жюри и собирает результаты проверки.

## Возможности

- Автоматический сбор ссылок на посты через API inkstory.net
- Парсинг текстов и авторов постов
- Два режима очереди: `balanced` (суммарная нагрузка) и `open` (общая очередь)
- AI-проверка текста через Groq API (орфография, грамматика, пунктуация)
- Планировщик: автозапуск парсера каждые N минут + финальный сбор в 23:55
- Панель администратора: верификация жюри, статистика, логи
- SQLite с WAL-режимом для безопасной параллельной работы
- Парсер работает в отдельном потоке — бот не зависает во время парсинга
- Экспорт итогов конкурса в Excel по дням

## Установка

```bash
# 1. Клонировать репозиторий
git clone https://github.com/Kit3ykpaini2000/inkstory-bot.git
cd inkstory-bot

# 2. Установить зависимости
pip install -r requirements.txt

# 3. Настроить переменные окружения
cp .env.example .env
# Открыть .env и заполнить BOT_TOKEN и GROQ_API_KEY

# 4. Инициализировать базу данных
python cli.py  # → выбрать «🗄 Инициализация БД»
```

## Настройка

Скопируй `.env.example` в `.env` и заполни:

```env
BOT_TOKEN=your_bot_token_here
QUEUE_MODE=balanced
GROQ_API_KEY=your_groq_api_key_here
```

| Переменная | По умолчанию | Описание |
|---|---|---|
| `BOT_TOKEN` | — | Токен бота от [@BotFather](https://t.me/BotFather) (**обязательно**) |
| `PARSER_INTERVAL` | `30` | Интервал автозапуска парсера в минутах |
| `PAGE_SIZE` | `20` | Постов на страницу при обходе API |
| `PAGE_PAUSE_LINKS` | `1.0` | Пауза между запросами при сборе ссылок (сек) |
| `PAGE_PAUSE_POSTS` | `1.5` | Пауза между запросами при парсинге постов (сек) |
| `QUEUE_MODE` | `balanced` | Режим очереди: `balanced` или `open` |
| `EXPIRE_MINUTES` | `30` | Через сколько минут незавершённый пост возвращается в очередь |
| `EXPIRE_CHECK_INTERVAL` | `5` | Как часто проверять просроченные посты (минуты) |
| `FINAL_PARSER_HOUR` | `23` | Час финального парсинга (по Москве) |
| `FINAL_PARSER_MINUTE` | `55` | Минута финального парсинга |
| `NEW_DAY_HOUR` | `0` | Час смены дня (по Москве) |
| `NEW_DAY_MINUTE` | `1` | Минута смены дня |
| `MAX_WORDS` | `100000` | Максимум слов при вводе жюри |
| `MAX_ERRORS` | `10000` | Максимум ошибок при вводе жюри |
| `GROQ_API_KEY` | — | Ключ от [console.groq.com](https://console.groq.com) |
| `GROQ_MODEL` | `llama-3.3-70b-versatile` | Модель Groq |
| `AI_CHUNK_SIZE` | `3000` | Максимум символов на один запрос к AI |

## Режимы очереди

| Режим | Описание |
|---|---|
| `balanced` | Пост назначается жюри с наименьшей суммарной нагрузкой (проверено + отклонено + в очереди) |
| `open` | Все посты в общей очереди — первый кто нажмёт `/next` тот и берёт |

В обоих режимах: если жюри взял пост но не проверил за `EXPIRE_MINUTES` минут — пост освобождается.

| 📅 Управление днями | Создание и управление днями конкурса |

| 📄 Просмотр логов | Последние строки лог-файла |

| 📤 Экспорт результатов | Экспорт итогов конкурса в Excel |
| 🗑 Полная очистка данных | Удаление всех постов и ссылок (жюри и дни сохраняются) |

```bash

sudo cp deploy/inkstory.service /etc/systemd/system/
sudo systemctl daemon-reload
sudo systemctl enable inkstory
sudo systemctl start inkstory

# Статус и логи
sudo systemctl status inkstory
journalctl -u inkstory -f
```

### Windows

```bat
# С консольным окном

# Без консольного окна (двойной клик)
deploy\start_silent.vbs
```

```bash
python scripts/export_results.py
```

## Утилиты

### fix_pending_queue.py

Если посты попали в `posts_info` со статусом `pending`, но запись в `queue` для них не создалась — этот скрипт исправляет ситуацию с учётом текущего `QUEUE_MODE`.

```bash
# Посмотреть что будет сделано (БД не трогает)
python scripts/fix_pending_queue.py --dry-run

# Применить
python scripts/fix_pending_queue.py

# Указать другую БД
python scripts/fix_pending_queue.py --db data/backup.db
```
