# inkstory-jury-tg-bot

Telegram-бот для организации литературного конкурса на [inkstory.net](https://inkstory.net).  
Парсит посты участников, распределяет их по жюри и собирает результаты проверки.

## Возможности

- Автоматический сбор ссылок на посты через API inkstory.net
- Парсинг постов на лету — текст не хранится в БД, загружается по требованию
- Два режима очереди: `balanced` (суммарная нагрузка) и `open` (общая очередь)
- AI-проверка текста через GitHub Models `gpt-4o-mini` (орфография, грамматика, пунктуация)
- Планировщик: автозапуск парсера каждые N минут + финальный сбор в 23:55
- **Telegram Mini App** — красивый веб-интерфейс для жюри и администраторов
- **FastAPI бэкенд** — REST API для Mini App
- **Туннель** — автоматический HTTPS туннель через `xtunnel` или `cloudflare` (бесплатно, без регистрации), URL обновляется в боте при каждом запуске
- Панель администратора: верификация жюри, статистика, логи, управление днями
- SQLite с WAL-режимом для безопасной параллельной работы
- Парсер работает в отдельном потоке — бот не зависает во время парсинга
- Экспорт итогов конкурса в Excel по дням

## Установка

```bash
# 1. Клонировать репозиторий
git clone https://github.com/Kit3ykpaini2000/inkstory-jury-tg-bot.git
cd inkstory-jury-tg-bot

# 2. Установить зависимости
pip install -r requirements.txt

# 3. Установить туннель

# Вариант A — xtunnel (рекомендуется):
# Скачать с https://xtunnel.ru и добавить в PATH

# Вариант B — cloudflare (регистрация не нужна):
# Linux (Debian/Ubuntu):
curl -L https://github.com/cloudflare/cloudflared/releases/latest/download/cloudflared-linux-amd64.deb -o cloudflared.deb
sudo dpkg -i cloudflared.deb
# Windows: скачать cloudflared-windows-amd64.msi с https://github.com/cloudflare/cloudflared/releases/latest

# 4. Настроить переменные окружения
cp .env.example .env
# Открыть .env и заполнить BOT_TOKEN и GROQ_API_KEY

# 5. Инициализировать базу данных
python cli.py  # → выбрать «🗄 Инициализация БД»

# 6. Добавить себя в жюри и верифицировать
python cli.py  # → «👥 Управление жюри» → «Добавить жюри» + «Верифицировать»

# 7. Создать день конкурса
python cli.py  # → «📅 Управление днями» → «Создать новый день»
```

## Настройка

Скопируй `.env.example` в `.env` и заполни:

```env
BOT_TOKEN=your_bot_token_here
QUEUE_MODE=balanced
GITHUB_TOKEN=your_github_token_here
```

| Переменная | По умолчанию | Описание |
|---|---|---|
| `BOT_TOKEN` | — | Токен бота от [@BotFather](https://t.me/BotFather) (**обязательно**) |
| `PARSER_INTERVAL` | `30` | Интервал автозапуска парсера в минутах |
| `PAGE_SIZE` | `20` | Постов на страницу при обходе API |
| `PAGE_PAUSE_LINKS` | `1.0` | Пауза между запросами при сборе ссылок (сек) |
| `PAGE_PAUSE_POSTS` | `1.5` | Пауза между запросами при парсинге постов (сек) |
| `QUEUE_MODE` | `balanced` | Режим очереди: `balanced` или `open` |
| `EXPIRE_MINUTES` | `30` | Через сколько минут незавершённый пост возвращается в очередь |
| `EXPIRE_CHECK_INTERVAL` | `5` | Как часто проверять просроченные посты (минуты) |
| `FINAL_PARSER_HOUR` | `23` | Час финального парсинга (по Москве) |
| `FINAL_PARSER_MINUTE` | `55` | Минута финального парсинга |
| `NEW_DAY_HOUR` | `0` | Час смены дня (по Москве) |
| `NEW_DAY_MINUTE` | `1` | Минута смены дня |
| `MAX_WORDS` | `100000` | Максимум слов при вводе жюри |
| `MAX_ERRORS` | `10000` | Максимум ошибок при вводе жюри |
| `GITHUB_TOKEN` | — | Personal Access Token от [github.com/settings/tokens](https://github.com/settings/tokens) |
| `GITHUB_MODEL` | `gpt-4o-mini` | Модель GitHub Models |
| `AI_CHUNK_SIZE` | `3000` | Максимум символов на один запрос к AI |

## Запуск

```bash
# Запустить бота + API + туннель (всё в одном)
python main.py

# Консольное управление (инициализация БД, экспорт, управление жюри и т.д.)
python cli.py
```

При запуске `main.py` автоматически:
1. Поднимается FastAPI на порту 8000
2. Запускается туннель (xtunnel или cloudflare, в зависимости от `TUNNEL_PROVIDER`)
3. URL Mini App обновляется в боте через Telegram API
4. Запускается бот с планировщиком

## Команды бота

| Команда | Описание |
|---|---|
| `/start` | Приветствие и список команд |
| `/register` | Регистрация жюри |
| `/next` | Получить следующий пост для проверки |
| `/stats` | Моя статистика |
| `/fullstats` | Общая статистика конкурса |
| `/admin` | Панель администратора |
| `/url` | *(только админ)* Обновить URL Mini App кнопки у всех жюри |
| `/tunnel` | *(только админ)* Перезапустить туннель и обновить URL |

Также в боте появляется кнопка **«Открыть панель»** — это Telegram Mini App с полным интерфейсом.

## Mini App

Mini App доступна через кнопку меню в боте. Включает три вкладки:

| Вкладка | Доступ | Описание |
|---|---|---|
| 📝 Проверка | Все верифицированные | Просмотр текста поста, принятие/отклонение/пропуск |
| 📊 Статистика | Все верифицированные | Личная и общая статистика конкурса |
| ⚙️ Админ | Только администраторы | Управление жюри, постами, днями, логи, экспорт |

## Режимы очереди

| Режим | Описание |
|---|---|
| `balanced` | Пост назначается жюри с наименьшей суммарной нагрузкой (проверено + отклонено + в очереди) |
| `open` | Все посты в общей очереди — первый кто нажмёт `/next` тот и берёт |

В обоих режимах: если жюри взял пост но не проверил за `EXPIRE_MINUTES` минут — пост освобождается.
